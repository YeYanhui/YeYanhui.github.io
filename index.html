注意区分硬链接和软连接的区别：

硬链接相当于重新再内存中备份一个文件，占用内存。
软连接不占用内存，只是引用。
区别:

  1. 源文件删除以后，软连接内容丢失。硬链接内容仍然存在。硬链接内容修改一次，所有的硬链接和软连接内容一起呗修改。
    软链接里修改内容相当于修改了源文件的内容。
  2. 硬链接是同一个文件内容，但是有多个文件名。

cat 将两个文件的内容重定向到一个文件中显示。

修改文件权限

chmod u/g/o/a +/-/ = rwx 文件
u - user 表示该文件的所有者
g - group 表示与该文件的所有者的属于同一组（group）者，即用户组。
o - other 表示其他以外的人
a - 表示这三者都是。

  • 增加权限
  • 撤销权限
    = 设定权限
    r : read 表示可读取，对于一个目录如果没有r权限，则不能通过ls 查看这个目录的内容。
    w : write 表示可写入，对于一个目录，如果没有w 权限,那么就意味着不能再目录下创建新的文件。
    x : excute 表示可执行，对于一个目录，如果没有x 权限，那么就意味着不能通过cd 进入这个目录。

数字法： rwx 这些权限也可以用数字来代替。
r :4, w:2, x :1, - 0(不具备任何权限)
7 : rwx 4+2+1 读写执行权限。

注意：如果想递归所有目录加上相同权限，需要加上参数“ -R ”。 如：chmod 777 test/ -R 递归 test 目录下所有文件加 777 权限

Mac 如何启动 ssh?

mac本身安装了ssh服务，默认情况下不会开机自启

1.启动sshd服务：
sudo launchctl load -w /System/Library/LaunchDaemons/ssh.plist

2.停止sshd服务：
sudo launchctl unload -w /System/Library/LaunchDaemons/ssh.plist

3查看是否启动：
sudo launchctl list | grep ssh

如果看到下面的输出表示成功启动了：
－－－－－－－－－－－－－－

  • 0 com.openssh.sshd

Mongodb 使用

  • db
    查看当前的数据库
  • show dbs /show databases
    查看当前所有数据库
  • use db
    切换数据库
  • db.dropDatabase()

集合的基础命令

db.createCollection(name,options)
db.createCollection(“stu”)
db.createCollection(“sub”,{capped:true, size:10})
capped:为true时，指定上限大小，当文档达到上限时，会将之前的数据覆盖，单位为字节。
查看集合： show collections;
删除集合： db.集合名称.drop()

数据类型

Object ID: 文档ID
String： 字符串，必须是有效的 utf-8
Boolean: 存储一个布尔值， true， 或false
Integer ： 整数可以是32位，或者64位，这取决于服务器。
Double:浮点值
Arrays： 数组列表
Object： 用于嵌入式文档，一个值为一个文档。
Null： 存储Null值
Timestamp: 时间戳， 总秒数。
Date： 存储当前日期或时间的Unix的时间格式。

查询

比较运算符

小于 $lt (less than)
小于等于： $lte (less than equal)
大于 ： $gt (greater than)
大于等于： $gte
不等于： $ne

db.stu.find({age:{$get:18}})

$or 或
查询命令：
db.stu.find({$or:[{age:18},{hometown:”桃花岛”}]})
db.stu.find({$or:[{age:{$gte:20}},{hometown:{$in:[“桃花岛”,”华山”]}}]})
db.stu.find({$where:function(){return this.age <= 18})

数据备份

mongodump -h dbhost -d dbname -o dbdirectory

-h: 服务器地址，也可以指定端口号
-d:需要备份的数据库名称
-o:备份的数据存放位置，此目录中存放着备份出来的数据。

mongorestore -h dbhost -d dbname –dir dbdirectory
-h: 服务器地址
-d: 需要恢复的数据库实例
–dir: 北非数据所在位置

mongorestore -h 192.168.196.128:27017 -d test1 -o ~/Desktop/test1/bak

聚合 aggregate 管道

每个文档通过一个由多个阶段组成的管道，可以对每个阶段的管道进行分组、过滤等功能，然后经过一系列的处理，输出相应的结果。
db.集合名称.aggregate({管道：{表达式}})

命令

$ group: 将集合中的文档分组，用于统计结果
$project ：修改输入文档的结构，如重命名，增加，删除字段，创建计算结果。
$ sort: 将输入文档排序后输出
$ skip: 跳过指定数量的文档，并返回余下的文档
$ unwind: 将数组类型的字段进行拆分

db.stu.aggregate(
{$group:{_id:”$gender”}}
)
db.sut.aggregate({$group:{_id:null,count:{$sum:1},mean_age:{$avg:”$age”}}})

db.stu.aggregate({$group:{_id:null,count:{$sum:1},mean_age:{$avg:”$age”}}},
{$progect:{gender:”$_id”}})

选择年龄大于20 的学生

$match
db.stu.aggregate(
{$match:{age:{$gt:20}}},
{$group:{_id:”$gender”,count:{$sum:1}}},
{$progect:{_id:0,gender:”$_id”,count:1}}
)

db.stu.aggregate(
{$match:{$or:[{age:{$gt:20}},{hometown:{$in:[“蒙古”,”大理”]}}]}},
{$group:{_id:”gender”,count:{$sum:1}}},
{$project:{_id:0,gender:”$_id,count:1”}}
）

db.t2.aggregate({$match:{username:”Alex”}},{$unwind:”$tags”})

db.t2.aggregate({$unwind:{path:”$size”,preserveNullAndEmptyArrays:true}})

验证码的识别

  • URL不变，验证码不变。

    • 请求验证码的地址，获取相应的验证码，识别

  • URL不变，验证码会变。

      2. 使用session请求登录页面，获取验证码的地址。

      3. 使用session请求验证码。识别。

      4. 使用session发送post请求。

      • URL不变，验证码会变
