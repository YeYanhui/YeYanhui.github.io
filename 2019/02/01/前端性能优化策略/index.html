前端性能优化策略

发表于 2019-02-01

1.前段性能优化
前段代码是下载到本地再开始运行。
2.后端代码写好以后再在服务器上运行。

代码部署

  1. 代码的压缩与合并
  2. 图片,js,css等静态资源使用和主站不同域名地址存储，从而使得在传输资源时不会带上不必要的cookie信息
  3. 使用内容分发网络CDN
  • 解释：数据从服务中心发送过来可能很慢，例如腾讯总部服务器在深圳，如果在其他地方也用深圳的服务器运行速度可能会很慢，如果我在北京，腾讯在北京设置一个服务器，我用CDN服务器分发，数据从北京服务器来进行传输，速度则会有明显的提高。
  1. 为文件设置Last——Modified、Expires(过期时间)和Etag（时间戳）.
  • 解释：1.如果当前时间和最后一次修改时间文件没有变化，则直接读取缓存，则实现了减少带宽，提高运行速度。2. 如果过期时间还没有到，我们则直接读取缓存，这样提高了网页运行效率，不用再重新下载和传输文件。从而实现不读服务器的数据。
  1. 使用GZIP压缩传送
  2. 权衡DNS 查找次数(使用不同域名会增加DNS查找)
  • 解释：域名解析为DNS，域名会先去访问一个DNS服务器，再把域名解析为一个IP地址服务器。然后再去访问IP地址服务器。同理：域名越多则会造成IP访问越多。2-3个就好，不要太多。否则会减慢运行速度。
  1. 避免不必要的重定向(加”/“)
  • 解析：加/ 就不用重定向了。

二. 编码

HTML

    1. 使用结构清晰，简单，语义化标签。
    2. 避免空的src和href.
      解析：避免寻找浪费
    3. 不要在HTML中缩放图片。
      解析：HTML中不缩放图片，则避免了渲染，显示更好看。

CSS

    1. 精简CSS选择器
      解析：最多四层，最好一层，不冲突。越短越好。
    2. 把CSS放到顶部
      解析：把CSS放到顶部，浏览器先读完CSS，再来渲染页面。一步到位。
      把CSS放到尾部，先渲染页面，再读CSS，再回来渲染页面。两步，多了一步。
    3. 避免@import方式引入样式。
    4. css中使用base64图片数据取代图片文件，减少请求数，在线base64网站:http://tool.css-js.com/base64.html
    5. 使用CSS动画来取代Javascript动画。
    6. 使用字体图片
    7. 使用CSS sprite(雪碧图)
    8. 使用SVG图形。
    9. 避免使用css表达式。
    10. 避免使用css滤镜。

Javascript

    1. 减少引用库的个数
    2. 使用requirejs或者seajs异步加载js
      解析： requirejs 和 seajs 是模块开发。类似于封闭函数。
    3. JS放到页面底部引入。
      解析： 加入新功能时，放在底部比较好。
    4. 避免全局查找。
    5. 使用原生方法。
      解析： 原生方法更快，但是容易不兼容。开发用jQuery。
    6. 用switch语句代替复杂的if else语句
    7. 减少语句数，比如说多个变量声明可以写成一句。
    8. 私用字面量表达式来初始化数组或者对象。
    9. 使用innerHTML来取代复杂的元素注入。
      解析： 尽量用字符串方式插入
    10. 使用事件代理（事件委托）
    11. 避免多次访问dom选择集。
    12. 高频触发事件设置使用函数节流。
    13. 使用web Storage 缓存数据。
