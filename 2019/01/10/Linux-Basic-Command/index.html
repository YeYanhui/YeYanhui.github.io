注意区分硬链接和软连接的区别：

硬链接相当于重新再内存中备份一个文件，占用内存。
软连接不占用内存，只是引用。
区别:

  1. 源文件删除以后，软连接内容丢失。硬链接内容仍然存在。硬链接内容修改一次，所有的硬链接和软连接内容一起呗修改。
    软链接里修改内容相当于修改了源文件的内容。
  2. 硬链接是同一个文件内容，但是有多个文件名。

cat 将两个文件的内容重定向到一个文件中显示。
cat 文件1 文件2 > 文件3

grep 搜索

grep ‘a’ 1.txt
grep -inv ‘^a’ 1.txt
i 忽略大小写，n 显示匹配行以及行号， -v 取反

find 命令

  • 查找当前目录下 大于4k小于 5 M 的文件
    find ./ -size +4k -size -5M
  • 查找当前目录下所有名为 test.s 文件
    find ./ -name test.s
  • 查找当前目录下所有名为 .py 文件
    find ./ -name ‘*.py’
  • 查找当前目录下权限为777的文件或者目录
    find ./ -perm 777
    归档管理： tar （tar只负责打包不负责压缩，压缩要用gzip）
  • 生成归档文件，创建打包文件。
  1. tar -c 文件名
  • 列出归档解档的详细过程，显示进度。
    tar -v 要压缩文件名 -f 压缩成的文件名
    tar -cvf 文件名.tar
  • gzip 文件压缩解压
    gzip -d 待解压文件 / 解压文件
    gzip -r 压缩所有子目录 / 压缩文件
  • 打包压缩一起
    tar -zcvf test.tar.gz *
  • 解压方法有所变化，解铃还须系铃人。
    tar -zxvf test.tar.gz
  • 解压到指定的目录 -C 后面加 指定的文件夹（大写字母C）
    tar -zxvf test.tar.gz -C ./c
  • 文件压缩解压: bzip2
    压缩用法：tar -jcvf 压缩包包名 文件 (tar jcvf bk.tar.bz2 *.c)
    解压用法： tar -jxvf 压缩包包名 （tar jxvf bk.tar.bz2）

zip方法

压缩文件： 1. zip -r 目标文件 源文件 2. zip 文件名 * ： 压缩当前所有文件
解压文件： unzip -d 解压后的目录 压缩文件

修改文件权限

chmod u/g/o/a +/-/ = rwx 文件
u - user 表示该文件的所有者
g - group 表示与该文件的所有者的属于同一组（group）者，即用户组。
o - other 表示其他以外的人
a - 表示这三者都是。

  • 增加权限
  • 撤销权限
    = 设定权限
    r : read 表示可读取，对于一个目录如果没有r权限，则不能通过ls 查看这个目录的内容。
    w : write 表示可写入，对于一个目录，如果没有w 权限,那么就意味着不能再目录下创建新的文件。
    x : excute 表示可执行，对于一个目录，如果没有x 权限，那么就意味着不能通过cd 进入这个目录。

数字法： rwx 这些权限也可以用数字来代替。
r :4, w:2, x :1, - 0(不具备任何权限)
7 : rwx 4+2+1 读写执行权限。

注意：如果想递归所有目录加上相同权限，需要加上参数“ -R ”。 如：chmod 777 test/ -R 递归 test 目录下所有文件加 777 权限

Ubuntu
whoami 查看当前用户
sudo -s: 切换到root

设置用户密码

sudo passwd 用户

退出登录账户： exit

  • 如果是图形界面，退出当前终端；
  • 如果是使用ssh远程登录，退出登陆账户；
  • 如果是切换后的登陆用户，退出则返回上一个登陆账号。

who 命令用于查看当前所有登录系统的用户信息。
-q 或 –count 只显示用户的登录账号和登录用户的数量。
-u 或者 –heading 显示列表题。

关机重启

reboot 重启
shutdown -r now 重新启动，shutdowm 会给用户提示。
shutdowm -h now 立刻关机，其中now 相当于时间为零的，立刻关机
shutdown -h 20:00
shutdown -h +10 10分钟后关机

软件安装和更新镜像源

sudo apt-get update 更新源

sudo apt-get install package 安装包

sudo apt-get remove package 删除包

sudo apt-cache search package 搜索软件包

sudo apt-cache show package 获取包的相关信息，如说明、大小、版本等

sudo apt-get install package –reinstall 重新安装包

sudo apt-get -f install 修复安装

sudo apt-get remove package –purge 删除包，包括配置文件等

sudo apt-get build-dep package 安装相关的编译环境

sudo apt-get upgrade 更新已安装的包

sudo apt-get dist-upgrade 升级系统

sudo apt-cache depends package 了解使用该包依赖那些包

sudo apt-cache rdepends package 查看该包被哪些包依赖

sudo apt-get source package 下载该包的源代码

sudo apt-get clean && sudo apt-get autoclean 清理无用的包

sudo apt-get check 检查是否有损坏的依赖

Mac 如何启动 ssh?

mac本身安装了ssh服务，默认情况下不会开机自启

1.启动sshd服务：
sudo launchctl load -w /System/Library/LaunchDaemons/ssh.plist

2.停止sshd服务：
sudo launchctl unload -w /System/Library/LaunchDaemons/ssh.plist

3查看是否启动：
sudo launchctl list | grep ssh

如果看到下面的输出表示成功启动了：
